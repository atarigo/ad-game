# 戰鬥系統

## 回合制設計

### 回合流程
每個回合包含以下階段，依序執行：

#### 1. 玩家控制階段 (PLAYER_CONTROL)
- 玩家可以執行的操作：
  - 使用道具
  - 使用技能
  - 移動角色位置
- 完成操作後點擊「結束回合」按鈕進入下一階段

#### 2. 玩家自動攻擊階段 (PLAYER_AUTO)
- 系統自動執行
- 我方所有存活角色攻擊正前方的敵人
- 如果正前方沒有敵人則跳過攻擊

#### 3. 小結算 (ENEMY_CHECK)
- 檢查敵人是否全滅
- 如果全滅 → 勝利
- 如果未全滅 → 進入敵人回合

#### 4. 敵人控制階段 (ENEMY_CONTROL)
- 敵人使用特定技能（未實作）
- 如果面前沒有我方單位，移動到有我方單位的列（未實作）

#### 5. 敵人自動攻擊階段 (ENEMY_AUTO)
- 系統自動執行
- 所有存活敵人攻擊正前方的我方單位

#### 6. 最終結算 (FINAL_CHECK)
- 檢查我方是否全滅
- 如果全滅 → 失敗
- 如果未全滅 → 回到玩家控制階段

## 戰鬥機制

### 攻擊計算
```
傷害 = max(1, 攻擊者ATK - 目標DEF)
```

### 目標選擇
- **正前方判定**：檢查是否在同一列
- 對於多格子單位：只要有任何一列重疊即視為在正前方
- 例：2x2 敵人占用第 2、3 列，我方在第 2 或第 3 列都可以攻擊

### 死亡處理
- 當 HP ≤ 0 時，單位死亡
- 死亡單位：
  - 從畫面移除
  - 從 ECS 世界移除
  - 不再參與戰鬥

## 勝利/失敗條件

### 勝利
- 敵人全滅（所有敵人 HP ≤ 0）

### 失敗
- 我方全滅（所有我方單位 HP ≤ 0）
